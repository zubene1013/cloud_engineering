Jdbc_강좌정리

JDBC ( Java DataBase Connectivity )

1. 개념

   - 자바프로그램언어와 DBMS 연동하는 기술로서 DMBS에 독립적임.

                    JDBC
    자바프로그램 <---------------------> DBMS(MySQL, Oralce, MS-SQL, DB2,... )

2. 특징
  - DBMS종류와 무관하게 일관된 방법으로 접근이 가능(DMBS에 독립적임)
    어떤 DBMS건 동일한 메서드로 접근한다.
  - JDBC는 비런타임 계열의 예외(SQLException)가 발생됨. 따라서 try~catch 또는 throws로 반드시 예외처리해야 됨.
  - JDBC는 기본적으로 auto commit임. ( 필요시 비활성 가능 ) 


3. 일관된 방법 구현?

   1)자바쪽에서 인터페이스 만듬
     내부에 추상메서드(ex. pubilc abstract void connect();
   2) 인터페이스를 DBMS 벤터에 제공
   3) 벤더에서 클래스를 만들때 인터페이스를 구현
      ex. public class Oracle implments Connection{
		pubilc  void connect(){}
           }
          public class MySQL implments Connection{
	       pubilc  void connect(){}
	  }
    
    4) 각 벤더마다 클래스파일들이 만들어지고, 이 클래스파일들(*.jar)과
       자바프로그램과 연동함.
       *.jar 파일을 드라이버(driver)라고 부름.

    5) 벤더에서 제공해준 드라이버(driver,jar파일)를
       자바프로그램에서 사용할 수 있도록 설정이 필요함.
