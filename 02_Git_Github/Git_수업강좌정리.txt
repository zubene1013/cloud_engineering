2. Git 개요

   - 버전 관리 시스템임
   - 리누스 토르발스가 만듬 ( 리눅스 창시자 )

   - 특징:
      로컬 저장소 및 원격 저장소 생성: git init 명령어
      로컬 저장소에 버전관리할 파일 생성 및 추가 : git add 명령어 ( 아직까지는 버전 생성 안됨 )
      버전관리할 파일의 버전 생성 :  git commit 명령어 + 커밋 메시지 반드시 설정( ex. "update 코드 추가" )
      버전 히스토리 확인 : git log 명령어
      버전간에 비교 : git diff 명령어
      특정버전으로 되돌아기 : git reset 명령어 , git revert 명령어
      새로운 브랜치 생성 ( 이전 브랜치에 영향을 미치지 않음 ) : git branch 브랜치명
      브랜치 병합: git merge 명령어
      로컬저장소에 브랜치를 원격 저장소에 보내기 가능: git push 명령어

6. 커밋 수정 ( --amend )
 
   -특징은 커밋을 만든 즉시 수정 가능하다.
    예> 포함해야될 파일이 누락되거나 커밋메세지 수정

   실습:
      $ git commit -m "some commit"
      $ touch world.txt 
      $ git add world.txt
      $ git commit --amend    # 자동으로 VSC 가 열리고 커밋메시지를 수정할 수 있게됨.
                                커밋메시지 수정후 저장하고 파일 close하면 됨.
      $ git log

7. 변경사항 비교

                             git diff 
      <----------------------------------------------------------------->

  작업디렉터리 -----------------------> 스테이지 ---------------------------> 저장소
                                       <------------------------------>    해시값3(HEAD)
				            git diff --staged              해시값2
                                                                           해시값1
	   <--------------------------------------------------------->
	          git diff HEAD (  git diff + git diff --staged )
 
  -git diff: 작업디렉터리와 저장소비교
  -git diff --staged : 스테이지와 저장소비교
  -git diff HEAD :  git diff + git diff --staged  모두 포함
  -git diff 해시값1  해시값2

   실습:
      $ mkdir git_diff
      $ cd git_diff
      $ git init   #.git 폴더 생성
      $ touch hello.txt
      $ echo "hello" >> hello.txt
      $ git add hello.txt
      $ git commit -m "first commit"

      # 작업디렉터리 수정
      $ echo "hello2" >> hello.txt

      # 작업디렉터리와 저장소비교
      $ git diff

      # 스테이지와 저장소 비교
      $ git add hello.txt
       git diff --staged

      # 작업디렉터리(hello3) + 스테이지(hello2) + 저장소(hello) 비교
      $ echo "hello3" >> hello.txt
      $ git diff HEAD

      # 해시값비교
      $ git commit -m "hello2 commit"
      $ git add hello.txt
      $ git commit -m "hello3 commit"

      $ git diff 37cf9b5 a7c793d

9. 작업 되돌리기

					  
  작업디렉터리 ---------------------> 스테이지 ------------------------> 로컬저장소
  hello.txt 수정                          <------------------------>
                                           git reset --soft HEAD^|해시값 ( git commit 되돌리기 )
		 
           <-------------------------------------------------------->
                          git reset --mixed HEAD^|해시값     ( git add 되돌리기 )

        <-------------------------------------------------------->
                          git reset --hard HEAD^|해시값      ( 작업디렉터리 수정전까지 되돌리기)

  # 커밋이전 상태 되돌리기
  git restore hello.txt          <== 작업디렉터리 작업 되돌리기
  git restore --staged hello.txt <== git add 되돌리기

  # 커밋이후 상태 되돌리기 ( 이전 커밋 히스토리 정보 삭제됨 )
   git reset --soft HEAD^|해시값 ( git commit 되돌리기 )
   git reset --mixed HEAD^|해시값     ( git add 되돌리기 )
   git reset --hard HEAD^|해시값      ( 작업디렉터리 수정전까지 되돌리기)

   실습:
     $ rm -rf git_reset
     $ mkdir git_reset
     $ cd git_reset
     $ git init
     $ touch hello.txt
     $ echo "hello" >> hello.txt
     $ git add hello.txt
     $ git commit -m "hello commit"

     $ echo "hello2" >> hello.txt
     $ git commit -am "hello2 commit"

     $ echo "hello3" >> hello.txt
     $ git commit -am "hello3 commit"

     # hello.txt 파일에 hello3 문자열 추가까지 되돌리기 
       즉 가장 마지막 커밋 삭제 
     $ git reset --hard HEAD^ 

     # git add 되돌리기 
     $ git reset --mixed HEAD^  

     $ git commit -am "hello2 commit" 

     # git commit 되돌리기
     $ git reset --soft HEAD^  

   실행결과:
       이전의 커밋정보가 삭제됨.
       이전 작업의 히스토리가 안남음.

1. 기본 브랜치

  - main 브랜치가 자동으로 제공 ( git init 할 때 )
  - 과거에는 master 브랜치가 기본이었음. 현재 사용 안함
  - 독립적으로 관리됨 (*****************)

2. 기본 명령어

   $ git branch : 브랜치 목록 보기
   $ git branch 브랜치명: 새로운 브랜치 생성
   $ git branch -M 브랜치명:  브랜치 이름 변경, 변경할 브랜치에서 작업
   $ git branch -D 브랜치명:  브랜치 강제 삭제, 다른 브랜치에서 작업
   $ git switch 브랜치명:  브랜치 변경
     git checkout 브랜치명: 브랜치 변경 (OLD )

   $ git switch -c 브랜치명 : 브랜치 생성하고 변경됨
   $ git log main..hotfix : main 기준으로 hotfix 차이점 출력
   $ git merge 브랜치명 : main에서 작업. main에 브랜치변경작업을 병합
   
   실습:
     $ mkdir git_branch
     $ cd git_branch
     $ git init
     $ touch hello.txt
     $ echo "hello" >> hello.txt
     $ git add .
     $ git commit -m "hello commit"

     $ git branch hotfix   # hotfix 이름의 새로운 브랜치 생성
	
     # main 브랜치에서 작업
     $ git branch -M master

     # main 브랜치에서 작업
     $ git branch -D hotfix

     # main 브랜치에서 hotfix로 변경
     $ git switch hotfix
       git checkout hotfix

     # feature 브랜치 만들면서 변경
     $ git switch -c feature

     # hotfix에 새로운 작업 추가 
       $ git switch hotfix
       $ echo "hello2" >> hello.txt
       $ git commit -am "hello2 commit"  <== main 과 hotfix가 다름.

     # main 기준으로 hotfix 차이점 출력
     $ git log main..hotfix

     # main에서 작업. main에 브랜치변경작업을 병합
     $ git merge  hotfix
       git branch -D hotfix  <== 병합후에는 hotfix 브랜치 삭제, git reflog 로 복구가능
-----------------------------------------------------------------------------------
3. 병합 종류

  1) Fast-forward ( 빨리감기 병합 )
      main브랜치에서 분기한 후(hotfix), hotfix에서 작업을 하고,main브랜치에서는 추가작업 없이 병합했을 때
      Fast-forward 방식으로 병합된다.

      실습:
       $ mkdir git_merge
         cd git_merge
	 git init
       
       #main작업
         touch hello.txt
	 echo "hello" >> hello.txt
	 git add .
	 git commit -m "hello commit" <--HEAD
	 git branch hotfix
	 git switch hotfix

       #hotfix작업
	  echo "hello2" >> hello.txt
	  git commit -am "hello2 commit" <--HEAD
	   git switch main
	  
       #main작업
          git merge hotfix  # Fast-forward 메시지 확인
	  cat hello.txt
	

  2) 병합커밋
    
      main 브랜치에서도 변경사항이 발생된 상태에서 병합방법으로 
      최종 커밋이 필요함.


    # 현재는 main과 hotfix간의 버전이 똑같음.

       #main작업
         touch world.txt
	 git add .
	 git commit -m "world commit"
	 git switch hotfix

      #hotfix작업
	  touch happy.txt
          git add .
  	  git commit -m "happy commit"
	  git switch main

       #main작업
          git merge hotfix   # 커밋해야되기 때문에 VSC 오픈되고 커밋메시지 작성후 close하면 병합커밋이 완료됨
	   Merge made by the 'ort' strategy.


  3) 병합충돌

      같은 파일의 같은라인을 서로 다르게 수정하고 병합할 때 발생됨.

      실습:

        mkdir git_conflict
	cd git_conflict
	git init

      #main작업
         touch hello.txt
	 echo "hello" >> hello.txt
	 git add .
	 git commit -m "hello commit" 
	 git branch hotfix
	 git switch hotfix  
       
      #hotfix작업
         vsc 열고 hello.txt 수정. hello2 로 변경하고 저장.
	 echo "hotfix" >> hello.txt
	 cat hello.txt
	   hello2
	   hotfix
	 git commit -am "hotfix commit"
	 git switch main

      #main작업
        echo "main" >> hello.txt
	cat hello.txt
	 hello
	 main
	git commit -am "main commit"

	git merge hotfix # 충돌발생, VCS열고 충돌직접 해결(코드수정)
	                  코드수정이 되었기 때문에 커밋해야 됨.

        git commit -am "final commit"
	cat hello.txt

Github 정리

1. 개요
   Git의 원격 저장소를 제공하는 클라우드 서비스임.
   저장소 기능 및 추가 기능도 제공됨. ( CI/CD 기능 지원: Github Actions )

2. github에 저장소 생성

  가. 회원가입 및 로그인 
  나. 상단의 repositories > New 선택 > 이름: github_test
  다. 로컬 저장소에서 원격저장소로 데이터를 올리는 방법(push) 가이드

      a. 로컬저장소가 아직 없는 경우
               echo "# github_test" >> README.md
		git init
		git add README.md
		git commit -m "first commit"
		git branch -M main # 브랜치이름 변경
		git remote add origin https://github.com/inky4832/github_test.git
		git push -u origin main

      b. 로컬저장소가 있는 경우
               git remote add origin https://github.com/inky4832/github_test.git
		git branch -M main
		git push -u origin main

   라. 원격브랜치 보기
	
       git branch  # 로컬 브랜치 보기
 
       git branch -r # 원격 브랜치 보기

       git branch -a # 로컬 + 원격 브랜치 보기

  마. 원격브랜치 삭제

     git push origin --delete  hotfix  
     
	만일 다음과 같이 오류가 발생했다면 다음과 같이 추가작업을 해야된다.
	error: unable to delete 'hotfix': remote ref does not exist
        error: failed to push some refs to 'https://github.com/inky4832/github_test.git'

        # 원격의 정보를 다시 가져옴
	git fetch -p origin
	git push origin --delete  hotfix

1. git clone 명령어

    원격저장소를 일반 디렉터리에 복사하는 작업.
    clone하면 디렉터리(원격저장소 이름)가 생기고 초기화(git init)까지 됨.
    원격저장소에 있는 모든 브랜치를 복사하지 않고 기본적으로 main만 복사됨.
    main 제외한 브랜치 복사는 추가작업이 필요함.


   문법:
       git clone http://~   : 로컬에 원격저장소명으로 디렉터리 생성
       git clone http://~   yyy_clone  : 지정된 로컬저장소명으로 디렉터리 생성

      c:\temp_clone

2. 원격의 hotfix 브랜치 clone 하기

      git checkout origin/hotfix
      git switch hotfix
