4. 데이터베이스 vs 데이터베이스 관리 시스템(DataBase Management System, DBMS )

   1) 데이터베이스(Database: DB )
    - 개인,회사, 관공서등에서 업무적으로 필요한 데이터 의미.
     예> 사원정보( 사원 DB )
        부서정보( 부서 DB )
	학생정보( 학생 DB )

   2) 데이터베이스 관리 시스템(DataBase Management System, DBMS )
     
     - 데이터베이스(DB)를 전문적이고 효율적으로 관리하기 위한 S/W.
      ex. MySQL, Oracle, MS-SQL, DB2,...

     -관리방법?
      SQL(Structured Query Language)문 이용. (DBMS 언어)

      SQL문 종류 (외우기)

      가. DML (데이터 조작어,  Data Manipulation Language)
         -INSERT문 ( 데이터 생성 )
	 -DELETE문 ( 데이터 삭제 )
	 -UPDATE문 ( 데이터 수정 )

      나. DDL (데이터 정의어, Data Definition Language)
        -CREATE문 ( 테이블생성)
	-ALTER문  ( 테이블수정)
	-DROP문   ( 테이블삭제)
	-TRUNCATE문 (테이블절삭)

      다. TCL ( 트랜잭션 관리 , Transaction Control Language )
         DTL ( Data Transaction Language)
       -COMMIT ( 트랜잭션 확정)
       -ROLLBACK (트랜잭션 취소)
       ==> DML과 연관있음

      라. DQL( Data Query Language)
       -SELECT문 ( 데이터 조회)

     ////////////////////
     CRUD 작업?
	 Create: 생성 ( insert 작업 의미 )
	 Read:   조회 ( select 작업 의미 ) 
	 Update: 수정 ( update 작업 의미 )
	 Delete: 삭제 ( delete 작업 의미 )
     //////////////////////


   - DBMS가 DB를 관리하는 방법?
     가. 관계형DB
     나. 계층형DB
     다. 망형DB
     라. 객체지향DB
     ...

6. SQL ( Structured Query Language )

 1)개요
  - RDB에서 사용하는 언어.
  - 식별자로 구성 ( 예약어 및 사용자 지정 식별자(ex. 테이블명, 컬럼명) 가능 )
  - 대소문자 구별 안함 ( SELECT와 select 동일 )
  - MySQL에서는 데이터 대소문자 구별안함 ( 'A'와 'a' 동일 )
    Oracle에서는 데이터 대소문자 구별함 ( 'A'와 'a' 다름 )

  - 자바에서는 문자와 문자열을 구별했음. ( 'A', "ABC" )
    SQL에서는 문자와 문자열을 구별 안함. ( 'A', 'ABC', '2026-12-13' )
    ""는 별칭(alias)에서만 사용하자.

  - 산술연산자 및 연산자
  - 함수( 자바의 메서드 역할 )
  - 조건문

 2)SQL문 종류1 ( RDB 종류에 따라서 )


    가. 표준화된 SQL문 ( 독립적인 SQL문, ANSI SQL )
      - 모든 RDBMS에서 사용 가능


    나. 벤더(업체)에 의존적인 SQL문
       -MySQL에서만 기
        Oracle에서만 사용
       -함수에서 주로 차이가 있음

 2)SQL문 종류2 ( 기능에 따라서 )-외우기

   가. DML ( Data Manipulation Language )
       - insert
       - update
       - delete
   나. DDL ( Date Definition Language )
       - create
       - alter
       - drop
       - truncate
   다. DQL ( Data Query Language )
       - select
   라. TCL(DTL)( Transaction Control Language, Data Transaction Language)
      - commit
      - rollback

 7. select 문

   1) 기능
      - 테이블 조회
      - 하나의 테이블 또는 여러 테이블 가능

   2) 기능 3가지

       -projection: 컬럼(열) 선택
       -selection : 레코드(행)선택
       -join      : 2개 이상의 테이블을 연결(조인)해서 원하는 컬럼(project)과 레코드(selection) 조회.
   
   3) 문법:

       select 절 // 컬럼명, 컬럼명2,...  <== 지정된 컬럼만 조회
		   * <== 모든컬럼
                   컬럼명+10, 컬럼명 * 1.1 <== 산술연산
		   함수명(컬럼명) <== ex. lower(ename)
		   distinct 컬럼명  <== 중복제거
		   리터럴 <== ex '홍길동'
		   컬럼명 as 별칭  <=== select할때 실행결과의 컬럼헤더값이 컬럼명이  아닌 별칭으로 보이게 됨.
                   컬럼명 as "별칭"    ex.   deptno as "department no" 
                   컬럼명 별칭          as 생략가능

       from 절;  // 테이블명

      
      ==> 실행결과는 테이블의 전체 레코드를 대상으로 조회됨.
         즉 테이블의 전체 레코드가 조회됨.(***********)

8. null 값

  1)개념
    -테이블의 컬럼에 값이 없음.
    -select 하면 비워져서 보임

  2)특징
    -기본적으로 모든 컬럼은 null값을 가질수 있음.
     강제적으로 null값을 허용하지 않도록 설정할 수도 있음.(NOT NULL 제약조건)

    - null값을 가진 컬럼을 연산하면 결과는 null로 나옴 (******************)
      따라서 null값을 임의의 기본값으로 변경하자.

      ifnull(컬럼명, 기본값) <== 컬럼명이 null일때 기본값으로 치환됨
                            oracle에서는 NVL(컬럼명, 기본값)

    - null값 찾기:  is null 연산자
                  is not null 연산자

    - 정렬시 null값을 가장 최소값으로 처리함. ( oracle에서는 최대값으로 처리함 )


9.제약조건 (Constraints)-외우기
 
 - 목적: 무결성 보장하는 방법
 - 컬럼단위로 지정

 1)primary key
   -기본키 제약조건
   -지정된 컬럼은 반드시 값을 가져야 되고(not null) 그 값은 유일한 값(unique)이어야 됨.
   -테이블 당 한개씩 지정 가능
   -용도: 레코드 식별 
        ex. emp테이블은 empno
	    dept테이블은 deptno
  - 복합컬럼 가능
  - 자동으로 index 생성됨.( ex. 책의 색인표 역할 )
    index이용하면 원하는 레코드를 빨리 찾을수 있음.
    하지만 index는 남용하면 안됨 ( 내부적으로 정렬 + B트리 작업이 추가되기 때문에 성능이 더 떨어질 수 있음 )

 2)unique
   - 유니크 제약조건
   - 지정된 컬럼은 반드시 유일한 값을 가짐. null값을 가질 수 있음(*********).
   - 테이블당 여러개 지정 가능
   - 복합컬럼 가능
    - 자동으로 index 생성됨.

 3)check
   - 비즈니스 로직 관련 제약조건임
   - 자바 프로그램으로 대체가능
    ex.  성별컬럼
	
	성별값: 남자/여자
	     남/여
	     F/M <======== 이것만 사용하도록 강제하도록 check 제약조건을 지정함.
	     Female/Male
	     1/0
        
	 중학생학년
	  1학년/2학년/3학년 <=== 이것만 가능하도록 check 제약조건을 지정함.
	  4학년

 4)foreign key
   - 참조키 또는 외래키 제약조건
   - 조인(join)할 때 사용됨.

     ex.
      
        emp ( slave 테이블)                 dept ( master 테이블 )
   ----------------------------           -----------
   empno ename ... sal  deptno            deptno   dname  loc
   (pk)                  (fk)              (pk)
   -----------------------------          ------------ 
   7369  SMITH     800    20               10    RESEARCH  NEW YORK
   7869  John      1500   20 ------------->20    SALES       ..
                                           30   
                          50(X)            40 
			  null(0)
   - 제약조건특징
     master테이블의 컬럼중에서 pk 또는 unique 컬럼만 참조할 수 있음.
     fk컬럼이 가질 수 있는 컬럼값은 master의 pk컬럼 또는 unique컬럼이 가지고 있는 값 또는 null값만 가질 수 있음.

   - master의 컬럼명과 fk컬럼명은 서로 달라도 무관 ( 권장은 동일하게 지정 )

  위 4개의 제약조건은 없던 기능을 추가(****)한 방식임.
  제약조건을 설정하는 방법 2가지를 모두 사용가능 ( 테이블레벨 방식, 컬럼레벨 방식 )
-------------------------------------------------------
  5)not null
   - null 값을 허용하지 않는 제약조건
   - 테이블당 여러개 지정 가능
   - 중복데이터 가능

  not null 은 null 허용된 기능을 허용하지 않도록 수정(****)한 방식임.
  제약조건을 설정하는 방법 2가지중 하나만 가능 (  컬럼레벨 방식 )


10. select + 조건식 지정

   1) 개념
     조건에 일치하는 레코드만 조회 ( selection 기능 )

   2) 문법

      select 절: *
                컬럼명,컬럼명,...
		컬럼명+10
		ifnull(컬럼명, 기본값)
		distinct 컬럼명
		컬럼명 as 별칭
      from  절 : 테이블명
      where 절 : 조건식

      실행순서:
         from > where > select 절

   3) 조건식에 사용가능한 연산자?

      가.비교연산자

        =  : 같냐?  (자바는 == 임 )

	!= : 같지않냐?
	>  : 크냐?
	>= : 크거나 같냐?
	<  : 작냐?
	<= : 작거나 같냐?

	ex.  select *  <== 3번
	     from emp <== 1번
	     where sal = 1000;  <== 2번

             where sal > 1000;
	     where sal <= 1000;
	     where sal != 1000;

	     where ename = 'SMITH';        //  (oracle에서는 항상''만 사용가능)
             where ename = 'smith';        //   MySQL에서는 소문자 가능 (oracle에서는 불가 )
	                                        
						Oracle에는 테이블명과 컬럼명은 소문자로 지정해도 대문자로 저장됨.
						값은 대소문자 구별해서 저장가능함.

	     where hiredate = '1980-12-17'; // 날짜데이터는 '' 지정해야됨.
	     where hiredate > '1980-12-17'; // 날짜데이터는 내부적으로 숫자로 관리되기 때문에 비교가 가능


  나. 범위 연산자
    
      between A  and  B : A와 B 범위에 존재하는 값을 찾음.
                          A와 B 포함됨


     ex.     select *  
	     from emp 
	     where sal between 1000 and 2000;

	     where hiredate between '1980-12-17' and '1984-12-17';

  다. 여러개 지정
    
      IN ( 값, 값2 , .. )


      ex.     select *  
	     from emp 
	     where sal IN (1000,2000,800);

	     where ename IN ('SMITH', 'FORD');
	     where hiredate IN ('1980-12-17', '1984-12-17');


 라. null 찾기

     is null
     is not null

     ex.    select *  
	     from emp 
	     where comm is null;      // 널 찾음
             where comm is not null;  // 널 아닌값 찾음

             where comm = null; // 안됨 


 마. like 연산자

    -패턴으로 조회

    -와일드카드(패턴)

      가. %  : 없거나 여러개 값으로 치환

      나. _ (언더바) : 반드시 하나의 값으로 치환

     
     ex.     select *  
	     from emp 
	     where ename like '%A%';   // A가 포함된 이름 검색

	     where ename like 'A%';    // A로 시작하는 이름 검색
          
	      where ename like '%A';    // A로 끝나는 이름 검색

	      where ename like '_A';    // 2글자이고 A로 끝나는 이름 검색


	      where ename like '_ _ _A%';    // 네번째 글자가 A인 이름 검색

 
 바. 논리연산자

     and :  그리고
     or  :  또는
     not :  부정

      ex.    ename이 SMITH 이고  sal 가 1000 보다 큰 사원?

             where ename ='SMITH' and sal > 1000;

	     ename이 SMITH 이거나  sal 가 1000 보다 큰 사원?

	     where ename ='SMITH' or sal > 1000;

	     ename이 SMITH 아닌 사원 ?

	      where ename != 'SMITH';

              where NOT ename = 'SMITH'

	      where NOT sal between 1000 and 2000;
	      where NOT sal IN (1000,2000,800);
	      where NOT ename like '_ _ _A%';


11. 정렬

   1)방법
     오름차순(기본): asc ( ascending )
     내림차순    : desc (descenging )

   2)문법;

      select 절  :
      from 절    :
      [where 절] :
      order by 절:  컬럼명, 별칭, select절에서 지정된 순서(1부터)

      ex.  select *
           from
	   order by 컬럼명;  //오름차순

	   order by 컬럼명 asc;  //오름차순

	   order by 컬럼명 desc;  //내림차순


	   select *
	   from em
	   order by sal;

	    order by sal asc;
	    order by sal desc;

	    order by hiredate desc;
 
  3) 다중정렬
       
        order by sal asc, deptno;   // sal 로 먼저 오름차순으로 정렬하고 sal이 같은 레코드는 deptno로 다시 오름차순으로 정렬

	order by sal desc, deptno;

	order by 1 desc, 3;

 5. 조건 함수
 https://docs.oracle.com/cd/E17952_01/mysql-8.0-en/flow-control-functions.html

   1) if

     IF(expr1,expr2,expr3): expr1 참이면 expr2
                                  거짓이면 expr3

      SELECT IF(1<2,'yes','no');
        -> 'yes'


   2) case ( 자바의 switch값(동등비교) 또는 다중 if(부등) 문과 비슷 )-ANSI

       가. 동등비교

          
              CASE value WHEN compare_value THEN result 
	                [WHEN compare_value THEN result ...] 
			[ELSE result] 
	      END

	 ex.  job에 따른 실수령 출력

	      CLERK      <== sal * 1.1
	      SALESMAN   <== sal * 1.2
	      MANAGER    <==  sal * 1.3
	      ANALYST     <==  sal * 1.4
	      이외          <== sal
	     
	    출력:
	        empno, ename, job,   sal , 실수령
		--------------------- -------
	                      CLERK   1000  1000*1.1
	                     SALESMAN 1000  1000*1.2

		
		select empno, ename, job, sal , case job when 'CLERK' THEN sal * 1.1
						         when 'SALESMAN' THEN sal * 1.2
							 when 'MANAGER' THEN sal * 1.3
							 when 'ANALYST' THEN sal * 1.4
							 else sal
		                                end as 실수령
		from emp;


       나. 부등비교 

               CASE WHEN condition THEN result 
	           [WHEN condition THEN result ...] 
		   [ELSE result] 
	       END
	       
	    ex. sal에 따른 직급출력
	
		 sal > 3000 <== 이사급
                 sal > 2500 <== 과장급
		 이외         <== 사원급

	      empno, ename, job,    sal , 직급
		--------------------- -------
	                      CLERK   4000  이사급
	                     SALESMAN 2800  과장급
			              800   사원급 


	   select empno, ename,job, sal,  case WHEN  sal > 3000 THEN '이사급'
	                                       WHEN  sal > 2500 THEN '과장급'
	                                       ELSE '사원급'
	                                  end as 직급
	   from emp;

1. 조인(join)

  1) query ( select ) 작업 3가지

     - projection: 컬럼 선택, select 절
     - selection: 레코드 선택, where 절
     - join : 2개 이상의 테이블에서 selection과 projection 처리


  2) 용도
     - 원하는 데이터가 여러 테이블에 분산되어 있는 경우에
       테이블을 연결(join)애서 데이터를 조회하는 방법.

  3) 종류1 (***************)

     가. inner 조인 ( 일반적으로 의미하는 조인 )
       - 테이블간의 연결고리(데이터가 일치)가 있는 데이터만 출력하는 방식.
         즉 일치하지 않으면 누락됨.

     나. outer 조인
        - inner 조인 + 누락된 데이터까지 포함해서 출력하는 방식

  4) 종류2

     가. ANSI 조인
        - DBMS 독립적

     나. 비 ANSI 조인
        - DBMS 의존적


  5) 종류3

    가. equi조인
      - 값이 정확하게 일치하는 방식으로 조인
      - 연산자로 사용하면 동등연산자 (=) 사용

    나. non-equi조인
      - 값이 정확하게 일치하지는 않는 방식으로 조인
      - 연산자로 사용하면 between A and B 사용 ( 부등연산자)

2. ANSI 조인

  1) cross 조인
     
     문법:
         select
	 from 테이블명1  cross join  테이블명2;


     동작:
         테이블명1의 하나의 레코드(행)와  테이블명2의 모든 레코드가 연결된다.
	 조인된 결과는 데이터로서 사용불가
	 최종 레코드는 테이블명1갯수 * 테이블명2갯수 만큼 결과가 출력됨.

  
  2) natural 조인
    
     문법1:
         select
	 from 테이블명1  natural join  테이블명2;

     문법2:
         select 별칭1.컬럼,  .., 별칭2.컬럼,...
	 from 테이블명1 별칭1 natural join  테이블명2 별칭2;


    동작: 테이블명1과 테이블명2에 공통컬럼을 찾아서 조인한다.
        만약 공통컬럼이 없으면 조인 불가.
	만약 공통컬럼이 2개라면 쌍으로 비교해서 조인.

	단점은 어떤 컬럼으로 조인되었는지를 추가로 확인하는 작업이 필요하다. ( 가독성이 떨어짐 )


    현재 조인조건:  natural join
    검색조건: where 절

     문법3:
         select 별칭1.컬럼,  .., 별칭2.컬럼,...
	 from 테이블명1 별칭1 natural join  테이블명2 별칭2
	 where 절;
       
   
  3) using 절

     문법1:
         select
	 from 테이블명1 [inner] join  테이블명2 using(공통컬럼명);
  
      문법2:
         select 별칭.컬럼,......
	 from 테이블명1 별칭  join  테이블명2 별칭 using(공통컬럼명); // using에서는 별칭사용 불가

       문법3:

	식
	 where 조건식

   4) on 절 (범용적)

    문법:

        select *

3. Outer 조인

   - 기본적으로 join은 inner 조인임( 일치하는 데이터만 출력됨)
     그런데 일치하지 않는 데이터까지 포함해서 조인하는 방식임.

  문법:

      select *
      from t1 LEFT OUTER JOIN t2 ON|USING 

       ==> t1 레코드는 모두 출력

      select *
      from t1 RIGHT OUTER JOIN t2 ON|USING

      ==> t2 레코드는 모두 출력

      # oracle에서는 LEFT|RIGHT|FULL  OUTER JOIN 지원

      
      ex.
          select *

5. 서브쿼리( subquery )

  1) 개념
     원하는 데이터를 한 번의 select만으로 안될때.
     여러번의 select문을 사용해서 원하는 결과를 얻음.

  ex. SMITH 사원보다 더 많은 sal를 받는 사원 출력?

      작업1> SMITH 사원의 sal 검색 ? 800 가정

           select sal
	   from emp
	   where ename ='SMITH';

      작업2>  select empno,ename,sal
           from emp
	   where sal > 800;


    최종적으로 작업1 과 작업2를 합치면 subquery 문법이 됨.

         최종 subquery문:


              -- 메인쿼리(main query)
	      select empno,ename,sal
              from emp
	      where sal > (select sal
	                   from emp
	                   where ename ='SMITH');


  2)특징
     - 서브쿼리는 반드시 ()를 가짐 
     - 서브쿼리가 먼저 실행되고 그 결과값을 이용해서 main query가 실행됨.
     - 서브쿼리가 사용가능한 위치는 거의 모든곳에 사용이 가능하고 여러번 사용 가능
       ex. where, having, from, insert, delete, update, create,...

      
    ex1.  emp테이블에서 사원번호가 7521 의 업무와 동일하고 급여가 7934보다 많은 사원의 정보(사원번호,이름,job,hiredate,sal)출력?
   
           -- 7521의job
	   select job
	   from emp
	   where empno = 7521;

	  -- 7934의sal
	  select sal
	   from emp
	   where empno = 7934;


	   select empno, ename,job,hiredate,sal
	   from emp
	   where job = (select job
	                 from emp
	                 where empno = 7521)

	         and
		 sal > (select sal
	                 from emp
	                where empno = 7934);
    
    
    
    ex2.  emp테이블에서 급여의 평균보다 적은 급여를 받는 사원 정보 출력(사원번호,이름,job,hiredate,sal)?

	   -- 급여의평균
	   select avg(sal)
	   from emp;

	   select empno, ename,job,hiredate,sal
	   from emp
	   where sal < (select avg(sal)
	                from emp);


    ex3.  emp테이블에서 20번 부서의 최소급여보다 많은 모든 부서를 출력? (부서명, 최소급여)

           -- 20번 부서의 최소급여
	   select min(sal)
	   from emp
	   where deptno=20;
	
	 
	  select deptno, min(sal)
	  from emp 
	  group by deptno
	  having min(sal) >  ( select min(sal)
	                       from emp
	                      where deptno=20);


    3) 연산자
      
      가. 단일행 연산자
         
	 - 서브쿼리가 실행된 후 하나의 행이(단일행) 리턴하는 경우에 사용가능한 연산자임
	 - 비교연산자 ( =, >, >=, <, <= )


      나. 복수행 연산자

         - 서브쿼리가 실행된 후 여러 행(복수행) 리턴하는 경우에 사용가능한 연산자임

    4) exists 연산자

      문법:
          select *
	  from 
	  where exists (서브쿼리);

      동작: 서브쿼리 실행결과가 있으면 main query가 실행되고

	ex.
	    select *
	    from emp
	    where exists (select ename
	                  from emp
			  where sal = 8000);
		

6. 인라인뷰 ( inline-view)

  개념:
      select
      from (서브쿼리) <== 인라인뷰

  목적: 성능 향상


  질문: emp 와 dept 테이블에서 부서별 sal총합 과 평균을 출력?

  일반:
          select e.deptno, sum(sal), round(avg(sal)), count(*)
	  from emp e join dept d ON e.deptno = d.deptno
	  group by deptno;

	   emp: 15개     dept: 4개


  인라인뷰:

        SELECT e.deptno, total_sum, total_avg, cnt
	FROM ( SELECT deptno, SUM(sal) total_sum, ROUND(AVG(sal)) total_avg, COUNT(*) cnt
	       FROM emp
	       GROUP BY deptno ) e JOIN dept d ON e.deptno = d.deptno;
	
	 인라이뷰: 4개    dept: 4개
