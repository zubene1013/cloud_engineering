4. 데이터베이스 vs 데이터베이스 관리 시스템(DataBase Management System, DBMS )

   1) 데이터베이스(Database: DB )
    - 개인,회사, 관공서등에서 업무적으로 필요한 데이터 의미.
     예> 사원정보( 사원 DB )
        부서정보( 부서 DB )
	학생정보( 학생 DB )

   2) 데이터베이스 관리 시스템(DataBase Management System, DBMS )
     
     - 데이터베이스(DB)를 전문적이고 효율적으로 관리하기 위한 S/W.
      ex. MySQL, Oracle, MS-SQL, DB2,...

     -관리방법?
      SQL(Structured Query Language)문 이용. (DBMS 언어)

      SQL문 종류 (외우기)

      가. DML (데이터 조작어,  Data Manipulation Language)
         -INSERT문 ( 데이터 생성 )
	 -DELETE문 ( 데이터 삭제 )
	 -UPDATE문 ( 데이터 수정 )

      나. DDL (데이터 정의어, Data Definition Language)
        -CREATE문 ( 테이블생성)
	-ALTER문  ( 테이블수정)
	-DROP문   ( 테이블삭제)
	-TRUNCATE문 (테이블절삭)

      다. TCL ( 트랜잭션 관리 , Transaction Control Language )
         DTL ( Data Transaction Language)
       -COMMIT ( 트랜잭션 확정)
       -ROLLBACK (트랜잭션 취소)
       ==> DML과 연관있음

      라. DQL( Data Query Language)
       -SELECT문 ( 데이터 조회)

     ////////////////////
     CRUD 작업?
	 Create: 생성 ( insert 작업 의미 )
	 Read:   조회 ( select 작업 의미 ) 
	 Update: 수정 ( update 작업 의미 )
	 Delete: 삭제 ( delete 작업 의미 )
     //////////////////////


   - DBMS가 DB를 관리하는 방법?
     가. 관계형DB
     나. 계층형DB
     다. 망형DB
     라. 객체지향DB
     ...

6. SQL ( Structured Query Language )

 1)개요
  - RDB에서 사용하는 언어.
  - 식별자로 구성 ( 예약어 및 사용자 지정 식별자(ex. 테이블명, 컬럼명) 가능 )
  - 대소문자 구별 안함 ( SELECT와 select 동일 )
  - MySQL에서는 데이터 대소문자 구별안함 ( 'A'와 'a' 동일 )
    Oracle에서는 데이터 대소문자 구별함 ( 'A'와 'a' 다름 )

  - 자바에서는 문자와 문자열을 구별했음. ( 'A', "ABC" )
    SQL에서는 문자와 문자열을 구별 안함. ( 'A', 'ABC', '2026-12-13' )
    ""는 별칭(alias)에서만 사용하자.

  - 산술연산자 및 연산자
  - 함수( 자바의 메서드 역할 )
  - 조건문

 2)SQL문 종류1 ( RDB 종류에 따라서 )


    가. 표준화된 SQL문 ( 독립적인 SQL문, ANSI SQL )
      - 모든 RDBMS에서 사용 가능


    나. 벤더(업체)에 의존적인 SQL문
       -MySQL에서만 기
        Oracle에서만 사용
       -함수에서 주로 차이가 있음

 2)SQL문 종류2 ( 기능에 따라서 )-외우기

   가. DML ( Data Manipulation Language )
       - insert
       - update
       - delete
   나. DDL ( Date Definition Language )
       - create
       - alter
       - drop
       - truncate
   다. DQL ( Data Query Language )
       - select
   라. TCL(DTL)( Transaction Control Language, Data Transaction Language)
      - commit
      - rollback

 7. select 문

   1) 기능
      - 테이블 조회
      - 하나의 테이블 또는 여러 테이블 가능

   2) 기능 3가지

       -projection: 컬럼(열) 선택
       -selection : 레코드(행)선택
       -join      : 2개 이상의 테이블을 연결(조인)해서 원하는 컬럼(project)과 레코드(selection) 조회.
   
   3) 문법:

       select 절 // 컬럼명, 컬럼명2,...  <== 지정된 컬럼만 조회
		   * <== 모든컬럼
                   컬럼명+10, 컬럼명 * 1.1 <== 산술연산
		   함수명(컬럼명) <== ex. lower(ename)
		   distinct 컬럼명  <== 중복제거
		   리터럴 <== ex '홍길동'
		   컬럼명 as 별칭  <=== select할때 실행결과의 컬럼헤더값이 컬럼명이  아닌 별칭으로 보이게 됨.
                   컬럼명 as "별칭"    ex.   deptno as "department no" 
                   컬럼명 별칭          as 생략가능

       from 절;  // 테이블명

      
      ==> 실행결과는 테이블의 전체 레코드를 대상으로 조회됨.
         즉 테이블의 전체 레코드가 조회됨.(***********)

8. null 값

  1)개념
    -테이블의 컬럼에 값이 없음.
    -select 하면 비워져서 보임

  2)특징
    -기본적으로 모든 컬럼은 null값을 가질수 있음.
     강제적으로 null값을 허용하지 않도록 설정할 수도 있음.(NOT NULL 제약조건)

    - null값을 가진 컬럼을 연산하면 결과는 null로 나옴 (******************)
      따라서 null값을 임의의 기본값으로 변경하자.

      ifnull(컬럼명, 기본값) <== 컬럼명이 null일때 기본값으로 치환됨
                            oracle에서는 NVL(컬럼명, 기본값)

    - null값 찾기:  is null 연산자
                  is not null 연산자

    - 정렬시 null값을 가장 최소값으로 처리함. ( oracle에서는 최대값으로 처리함 )


9.제약조건 (Constraints)-외우기
 
 - 목적: 무결성 보장하는 방법
 - 컬럼단위로 지정

 1)primary key
   -기본키 제약조건
   -지정된 컬럼은 반드시 값을 가져야 되고(not null) 그 값은 유일한 값(unique)이어야 됨.
   -테이블 당 한개씩 지정 가능
   -용도: 레코드 식별 
        ex. emp테이블은 empno
	    dept테이블은 deptno
  - 복합컬럼 가능
  - 자동으로 index 생성됨.( ex. 책의 색인표 역할 )
    index이용하면 원하는 레코드를 빨리 찾을수 있음.
    하지만 index는 남용하면 안됨 ( 내부적으로 정렬 + B트리 작업이 추가되기 때문에 성능이 더 떨어질 수 있음 )

 2)unique
   - 유니크 제약조건
   - 지정된 컬럼은 반드시 유일한 값을 가짐. null값을 가질 수 있음(*********).
   - 테이블당 여러개 지정 가능
   - 복합컬럼 가능
    - 자동으로 index 생성됨.

 3)check
   - 비즈니스 로직 관련 제약조건임
   - 자바 프로그램으로 대체가능
    ex.  성별컬럼
	
	성별값: 남자/여자
	     남/여
	     F/M <======== 이것만 사용하도록 강제하도록 check 제약조건을 지정함.
	     Female/Male
	     1/0
        
	 중학생학년
	  1학년/2학년/3학년 <=== 이것만 가능하도록 check 제약조건을 지정함.
	  4학년

 4)foreign key
   - 참조키 또는 외래키 제약조건
   - 조인(join)할 때 사용됨.

     ex.
      
        emp ( slave 테이블)                 dept ( master 테이블 )
   ----------------------------           -----------
   empno ename ... sal  deptno            deptno   dname  loc
   (pk)                  (fk)              (pk)
   -----------------------------          ------------ 
   7369  SMITH     800    20               10    RESEARCH  NEW YORK
   7869  John      1500   20 ------------->20    SALES       ..
                                           30   
                          50(X)            40 
			  null(0)
   - 제약조건특징
     master테이블의 컬럼중에서 pk 또는 unique 컬럼만 참조할 수 있음.
     fk컬럼이 가질 수 있는 컬럼값은 master의 pk컬럼 또는 unique컬럼이 가지고 있는 값 또는 null값만 가질 수 있음.

   - master의 컬럼명과 fk컬럼명은 서로 달라도 무관 ( 권장은 동일하게 지정 )

  위 4개의 제약조건은 없던 기능을 추가(****)한 방식임.
  제약조건을 설정하는 방법 2가지를 모두 사용가능 ( 테이블레벨 방식, 컬럼레벨 방식 )
-------------------------------------------------------
  5)not null
   - null 값을 허용하지 않는 제약조건
   - 테이블당 여러개 지정 가능
   - 중복데이터 가능

  not null 은 null 허용된 기능을 허용하지 않도록 수정(****)한 방식임.
  제약조건을 설정하는 방법 2가지중 하나만 가능 (  컬럼레벨 방식 )


10. select + 조건식 지정

   1) 개념
     조건에 일치하는 레코드만 조회 ( selection 기능 )

   2) 문법

      select 절: *
                컬럼명,컬럼명,...
		컬럼명+10
		ifnull(컬럼명, 기본값)
		distinct 컬럼명
		컬럼명 as 별칭
      from  절 : 테이블명
      where 절 : 조건식

      실행순서:
         from > where > select 절

   3) 조건식에 사용가능한 연산자?

      가.비교연산자

        =  : 같냐?  (자바는 == 임 )

	!= : 같지않냐?
	>  : 크냐?
	>= : 크거나 같냐?
	<  : 작냐?
	<= : 작거나 같냐?

	ex.  select *  <== 3번
	     from emp <== 1번
	     where sal = 1000;  <== 2번

             where sal > 1000;
	     where sal <= 1000;
	     where sal != 1000;

	     where ename = 'SMITH';        //  (oracle에서는 항상''만 사용가능)
             where ename = 'smith';        //   MySQL에서는 소문자 가능 (oracle에서는 불가 )
	                                        
						Oracle에는 테이블명과 컬럼명은 소문자로 지정해도 대문자로 저장됨.
						값은 대소문자 구별해서 저장가능함.

	     where hiredate = '1980-12-17'; // 날짜데이터는 '' 지정해야됨.
	     where hiredate > '1980-12-17'; // 날짜데이터는 내부적으로 숫자로 관리되기 때문에 비교가 가능


  나. 범위 연산자
    
      between A  and  B : A와 B 범위에 존재하는 값을 찾음.
                          A와 B 포함됨


     ex.     select *  
	     from emp 
	     where sal between 1000 and 2000;

	     where hiredate between '1980-12-17' and '1984-12-17';

  다. 여러개 지정
    
      IN ( 값, 값2 , .. )


      ex.     select *  
	     from emp 
	     where sal IN (1000,2000,800);

	     where ename IN ('SMITH', 'FORD');
	     where hiredate IN ('1980-12-17', '1984-12-17');


 라. null 찾기

     is null
     is not null

     ex.    select *  
	     from emp 
	     where comm is null;      // 널 찾음
             where comm is not null;  // 널 아닌값 찾음

             where comm = null; // 안됨 


 마. like 연산자

    -패턴으로 조회

    -와일드카드(패턴)

      가. %  : 없거나 여러개 값으로 치환

      나. _ (언더바) : 반드시 하나의 값으로 치환

     
     ex.     select *  
	     from emp 
	     where ename like '%A%';   // A가 포함된 이름 검색

	     where ename like 'A%';    // A로 시작하는 이름 검색
          
	      where ename like '%A';    // A로 끝나는 이름 검색

	      where ename like '_A';    // 2글자이고 A로 끝나는 이름 검색


	      where ename like '_ _ _A%';    // 네번째 글자가 A인 이름 검색

 
 바. 논리연산자

     and :  그리고
     or  :  또는
     not :  부정

      ex.    ename이 SMITH 이고  sal 가 1000 보다 큰 사원?

             where ename ='SMITH' and sal > 1000;

	     ename이 SMITH 이거나  sal 가 1000 보다 큰 사원?

	     where ename ='SMITH' or sal > 1000;

	     ename이 SMITH 아닌 사원 ?

	      where ename != 'SMITH';

              where NOT ename = 'SMITH'

	      where NOT sal between 1000 and 2000;
	      where NOT sal IN (1000,2000,800);
	      where NOT ename like '_ _ _A%';


11. 정렬

   1)방법
     오름차순(기본): asc ( ascending )
     내림차순    : desc (descenging )

   2)문법;

      select 절  :
      from 절    :
      [where 절] :
      order by 절:  컬럼명, 별칭, select절에서 지정된 순서(1부터)

      ex.  select *
           from
	   order by 컬럼명;  //오름차순

	   order by 컬럼명 asc;  //오름차순

	   order by 컬럼명 desc;  //내림차순


	   select *
	   from em
	   order by sal;

	    order by sal asc;
	    order by sal desc;

	    order by hiredate desc;
 
  3) 다중정렬
       
        order by sal asc, deptno;   // sal 로 먼저 오름차순으로 정렬하고 sal이 같은 레코드는 deptno로 다시 오름차순으로 정렬

	order by sal desc, deptno;

	order by 1 desc, 3;

2. 문자열 함수
   https://docs.oracle.com/cd/E17952_01/mysql-8.0-en/string-functions.html

  
    1)대문자/소문자
     
       lower(ename)
       upper(ename)
    
    2) 연결
       CONCAT(str1,str2,...)
       SELECT CONCAT('My', 'S', 'QL');
        -> 'MySQL'

       CONCAT_WS(separator,str1,str2,...)
       SELECT CONCAT_WS(',', 'First name', 'Second name', 'Last Name');
        -> 'First name,Second name,Last Name'
      
   3) 특정문자채우기
     
        ex.  'hello' -----------> 'hello*****'
	                          '*****hello'  

	LPAD(str,len,padstr)
	SELECT LPAD('hi',4,'??');
        -> '??hi'

	RPAD(str,len,padstr)
	SELECT RPAD('hi',5,'?');
        -> 'hi???'

    4)부분열
       SUBSTR 와 SUBSTRING는 동의어(synonym)임

       SUBSTR(str,pos,len)   # pos는 1부터.
       SUBSTRING(str,pos,len)

       SELECT SUBSTRING('Quadratically',5);
        -> 'ratically'

       LEFT(str,len)
        SELECT LEFT('foobarbar', 5);
        -> 'fooba'

       RIGHT(str,len)
        SELECT RIGHT('foobarbar', 4);
        -> 'rbar'

    5) 길이

       LENGTH(str) : 지정된 str의 byte 갯수
                    영문자는 1byte
		    한글은 3byte로 처리됨

       CHAR_LENGTH(str): 지정된 str의 갯수
         
    6) 치환
      
       REPLACE(str,from_str,to_str)
        SELECT REPLACE('www.mysql.com', 'w', 'Ww');
        -> 'WwWwWw.mysql.com'

       INSERT(str,pos,len,newstr)
       SELECT INSERT('Quadratic', 3, 4, 'What');
        -> 'QuWhattic'

    7) 특정문자의 위치
       
         없으면  0 반환 ( 자바의 indexOf는 -1 반환 )

         INSTR(str,substr) 

	 SELECT INSTR('foobarbar', 'bar');
           -> 4
	mysql> SELECT INSTR('xbar', 'foobar');
          -> 0


    8) 공백제거
	
	왼쪽공백제거 : LTRIM(str)
	오른쪽공백제거: RTRIM(str)
	양쪽공백제거:  TRIM(str)
         
    9) 특정문자 제거

        TRIM(BOTH 삭제값 FROM str) : 양쪽에서 특정 문자 삭제
	TRIM(LEADING 삭제값 FROM str) : 왼쪽에서 특정 문자 삭제
	TRIM(TRAINIG 삭제값 FROM str) : 오른쪽에서 특정 문자 삭제

    10) 반복 출력
      
        REPEAT(str,count)
	SELECT REPEAT('MySQL', 3);
        -> 'MySQLMySQLMySQL'

    11) 거꾸로
        
	REVERSE(str)
	SELECT REVERSE('abc');
        -> 'cba'

    12) 공백반환
        
	 SPACE(N)
	 SELECT SPACE(6);
        -> '      '

    13) 정수값을 포맷을 가진 문자열로 출력

        FORMAT(정수,소수점자릿수[,locale])

	SELECT FORMAT(12332.123456, 4);
        -> '12,332.1235'

	SELECT FORMAT(129999332.123456, 0);
	-> '12,332'

	SELECT FORMAT(12332.123456, 4, 'ko_kr');  # 기본 locale: en_us
        -> '12,332.1235'


3. 숫자함수
https://docs.oracle.com/cd/E17952_01/mysql-8.0-en/numeric-functions.html

  1)절대값

     ABS(X)
     SELECT ABS(2);
        -> 2
     SELECT ABS(-32);
        -> 32
 
  2) 부호
   
     SIGN(X)

     양수: 1
     0:  0
     음수: -1

     SELECT SIGN(-32);

     SELECT SIGN(0);
        -> 0
     SELECT SIGN(234);
        -> 1
     
  3) 반올림

     ROUND( 값, [자릿수])

     SELECT ROUND(150.000,2), ROUND(150,2);


  4) 절삭 (버림)

      TRUNCATE(값, 자릿수)

      SELECT TRUNCATE(1.223,1);
       -> 1.2

      SELECT TRUNCATE(1.999,0);
        -> 1
    
  5) ceil, floor

     ==> 소수점을 가진 값을 정수로 변경

     ceil: 지정된 값과 같거나 큰 정수 리턴
      
      SELECT CEILING(1.23);
        -> 2
    
     floor: 지정된 값과 같거나 작은 정수 리턴
      SELECT FLOOR(1.23), FLOOR(-1.23);
        -> 1, -2

   6) 나머지

        -> 1


4. 날짜 함수 ( 비 ANSI 문법 )
 
   1) 현재날짜와 시간
     
       NOW() : 기본포맷은 'YYYY-MM-DD hh:mm:ss'
               시간을 구하는 시점이 SQL문이 실행시 시간을 구함.

	       current_timestamp(), current_timestamp


       SYSDATE(): 기본포맷은 'YYYY-MM-DD hh:mm:ss'
               시간을 구하는 시점이 함수호출시 시간을 구함.

   2) 현재날짜 
       
       curdate(), current_date(), current_date
       
       SELECT CURDATE(), current_date(), current_date;
        -> '2008-06-13'

   3) 현재시간

       CURTIME(), CURRENT_TIME(), CURRENT_TIME

       SELECT CURTIME() , CURRENT_TIME(), CURRENT_TIME;
	+-----------+
	| CURTIME() |
	+-----------+
	| 19:25:37  |


    4) 날짜 더하기 
     
         ADDDATE(date,days) <== 지정된 날짜에 day 더하기 , DATE_ADD() 는 동의어
         
	 SELECT ADDDATE('2008-01-02', 31);
        -> '2008-02-02'

         ADDDATE(date,INTERVAL expr unit) # unit 단위: YEAR|MONTH|DAY|HOUR|MINUTE
         
	 SELECT DATE_ADD('2008-01-02', INTERVAL 31 DAY);
         -> '2008-02-02'
         
	 SELECT DATE_ADD('2008-01-02', INTERVAL 31 YEAR);
         SELECT DATE_ADD('2008-01-02', INTERVAL 31 MONTH);
 

     5) 날짜 빼기

          SUBDATE(expr,days) <== 지정된 날짜에 day 빼기, DATE_SUB() 동의어

	  SELECT SUBDATE('2008-01-02 12:00:00', 31);
        -> '2007-12-02 12:00:00'

	 
	  SUBDATE(date,INTERVAL expr unit) # unit 단위: YEAR|MONTH|DAY|HOUR|MINUTE

	  SELECT DATE_SUB('2008-01-02', INTERVAL 31 DAY);
        -> '2007-12-02'

	  
	  SELECT DATE_SUB('2008-01-02', INTERVAL 31 YEAR);

	  SELECT DATE_SUB('2008-01-02', INTERVAL 31 MONTH);

	 
    6) 날짜 차이
      
      DATEDIFF(expr1,expr2) #  expr1 - expr2
      SELECT DATEDIFF('2007-12-31 23:59:59','2007-12-30');
        -> 1


      TIMESTAMPDIFF(unit,datetime_expr1,datetime_expr2) # unit 단위: YEAR|MONTH|DAY|HOUR|MINUTE
      SELECT TIMESTAMPDIFF(MONTH,'2003-02-01','2003-05-01');
        -> 3
      SELECT TIMESTAMPDIFF(YEAR,'2002-05-01','2001-01-01');
        -> -1
      SELECT TIMESTAMPDIFF(MINUTE,'2003-02-01','2003-05-01 12:05:55');
        -> 128885
